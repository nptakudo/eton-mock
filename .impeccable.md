# Eton — Design Context

## Design Context

### Users

**Primary**: Pedagogy coaches, instructional leads, and centre directors at multi-campus early-childhood schools (EtonHouse-scale). They sit between the teacher and the teaching framework — they're the person who watches a recording, maps what they see to EYLF or QTT, and hands a written report back to a teacher by end of week.

**Context of use**: Desktop-first, during focused work hours. They open a report to review it, annotate it, and share it. They are not skimming — they read carefully and scrutinise claims against evidence. This is not a phone-between-meetings tool.

**Job to be done**: Turn a classroom recording into an evidence-backed, framework-mapped observation report that a teacher can read without feeling judged. The coach is not an inspector; the tool must reflect that.

**Emotional goal**: The interface should make them feel like a careful professional using a precise instrument — not like a user of a SaaS tool. The report itself is the deliverable. The interface is the workbench.

---

### Brand Personality

**Three words**: Warm · Observant · Non-judgmental

- *Warm*: The language is coaching, not evaluation. Sections are "Teacher Moves That Landed" and "Where the Practice Can Tighten" — not "Strengths" and "Weaknesses." The interface must match that register.
- *Observant*: Every claim is grounded in a timestamp, a speaker label, and a direct quote. The design should honour that precision — citations are first-class objects, not footnotes to hide.
- *Non-judgmental*: Prompt rules literally say "Keep the tone constructive, not evaluative" and "Do not infer intent — only describe behaviour." The interface should never feel like a scoring or grading system.

---

### Aesthetic Direction

**Theme**: Light-only. The viewing context is a desk or classroom office during the day, not a dark terminal. The current cream palette (`#faf9f8`, `#f0efed`) is directionally right but can be pushed further if something more distinctive earns its place.

**Visual tone**: Quiet, precise, document-like — closer to reading a well-edited memo or a research paper with clickable footnotes than using a SaaS dashboard. The report IS the product; the chrome around it should recede.

**Current palette is a starting point** — open to replacement if something more distinctive fits better. The key constraint: no clinical white, no dashboard grey. Surfaces should feel like paper, not screens.

**Type**: Currently system fonts only. Web fonts can be introduced if they serve the aesthetic — no hard constraint. Preference: something that reads as considered and editorial, not "app-y." Avoid the reflex_fonts_to_reject list entirely.

**References**:
- **Granola** — the soft-pink chat-box border, cream background, framework chips, "What did we learn?" placeholder. Direct DNA.
- **Notion** — collapsible sidebar with chevron-on-hover, breadcrumb topbar, document-as-canvas main column, "/" prompt syntax.
- **Linear** — keyboard-affordance labels, tab badges with counts, restrained 1.5px stroke iconography, modal restraint.
- **Stripe Docs / Vercel** — report body hierarchy, generous whitespace, citations as first-class objects.
- **Readwise Reader** — evidence-card pattern: timestamp + speaker chip + quote + reason.

**Anti-references** (must not look like):
- Compliance / inspection tools — no traffic-light scoring, no red/amber/green badges, no progress rings, no "Score: 7.2/10"
- Typical EdTech SaaS — no purple gradients, no cartoon mascots, no oversized hero illustrations, no "🎉 Great job!" empty states
- Meeting-AI clones (Otter, Fireflies, Fathom) — no waveform visualisations, no speaker-talk-time donut charts, no "AI insights" sparkle icons
- Google Classroom / LMS — no card-grid assignment dashboards, no subject-and-grade left rails
- Generic chat UI (ChatGPT, Claude.ai) — the chat input is a side-tool, not the product; it must not dominate
- Slack / enterprise grey — no `#f5f5f5`, no blue primary buttons everywhere, no Inter-on-white at 13px

---

### Design Principles

1. **The report is the product.** Every design decision should serve readability of the observation report. Navigation, chat, and controls are supporting infrastructure — they should recede, not compete.

2. **Citations earn their place.** Evidence is anchored to timestamps, speakers, and direct quotes. The design should make inline citations visible and hoverable, not a footnote afterthought. Evidence cards are first-class, not collapsed by default.

3. **Coaching tone, not inspection tone.** No scoring visuals, no red/green status signals on content. Warm graphite on cream. If it could appear on an inspection dashboard, it's wrong for Eton.

4. **Document before dashboard.** Typography, line-height, and column width take precedence over data density. The page should read like a carefully typeset document, not a metrics view.

5. **Restraint is the differentiator.** The current palette is nearly shadowless, nearly borderless, nearly accent-free. That's the aesthetic — protect it. Every new element added should justify its visual weight explicitly.
